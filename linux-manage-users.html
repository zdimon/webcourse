Manage with users in Linux.

/etc/passwd

Contains one line for each user account.

Username:Password:User ID:Group ID:User ID Info:Home directory:Command/shell

Password - An x character indicates that encrypted password is stored in /etc/shadow file.

User ID (UID):

0 (zero) is reserved for root and UIDs

1-99 are reserved for other predefined accounts.

100-999 are reserved by system for administrative and system accounts/groups.

Group ID (GID): The primary group ID (stored in /etc/group file)

/etc/passwd file permission

ls -l /etc/passwd

Directory permission

ls -ld zdimon

Get information

grep zdimon /etc/passwd

Adding user

adduser <username>

Change user

su <username>

Ctr+d - to get back

Add user to sudores

sudo adduser <username> sudo

Add user to group

useradd -G <> username

This will create a new user and add it to group.

usermod -a -G zdimon test

Changing permissions using chmod

read (r'), write (w'), and execute (`x')

 chmod u=rx file

(Give the owner rx permissions, not w)

 chmod go-rwx file

(Deny rwx permission for group, others)

 chmod g+w file

(Give write permission to the group)

 chmod a+x file1 file2

(Give execute permission to everybody)

 chmod g+rx,o+x file

(OK to combine like this with a comma)

Deleting user

deluser --remove-home userName

Locale issues

sudo locale-gen "en_US.UTF-8"
sudo locale-gen "ru_RU.UTF-8"
